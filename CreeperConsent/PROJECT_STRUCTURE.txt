REQUIREMENTS FULFILLMENT
═════════════════════════════════════════════════════════════════════════════

Core Behavior:
  ✅ Detects creeper explosion prime (ExplosionPrimeEvent)
  ✅ Shows full-screen GUI popup immediately
  ✅ Displays: "A creeper has asked for permission to explode"
  ✅ Two buttons: "Sure, let's go" (accept) and "Nah, not today" (deny)
  ✅ Player hangs indefinitely if no response
  ✅ No response persistence between prompts

Configuration:
  ✅ enabled: true/false toggle
  ✅ Customizable message strings
  ✅ Default config.yml provided
  ✅ Configuration reloadable on restart

GUI Implementation:
  ✅ 9-slot chest inventory (clean, readable)
  ✅ Colored wool buttons (lime = yes, red = no)
  ✅ Filler blocks in remaining slots
  ✅ Custom formatted text on buttons

Technical:
  ✅ Paper 1.21.1 target
  ✅ Java 21 compatible
  ✅ Only prompts triggering player
  ✅ Handles null/offline players gracefully
  ✅ Multi-player support
  ✅ No external dependencies

QUICK START
═════════════════════════════════════════════════════════════════════════════

1. BUILD
   $ mvn clean package
   Output: target/CreeperConsent-1.0.0.jar

2. DEPLOY
   $ cp target/CreeperConsent-1.0.0.jar /path/to/server/plugins/
   $ cd /path/to/server && ./start.sh

3. TEST
   /summon minecraft:creeper
   [Hit creeper with sword]
   [GUI should appear!]

4. VERIFY
   Check logs: [CreeperConsent] CreeperConsent enabled!
